System Blueprint — Business Logic, State Mapping & Mode Enforcement
Trust Score Formula
TradeMatch Score = (Vetting × 0.30) + (Sentiment × 0.25) + (ShowUpScore × 0.25) + (Reliability × 0.20) ShowUpScore = (Confirmed ÷ Scheduled) × 100 // Confirmed = check-in within 90min window // Min 3 visits before score displays Dispute penalty: −5 pts per upheld case Insurance lapse: Vetting pillar → 0
Arrival PIN Flow
1HO enables Mandatory PIN in Settings → 4-digit PIN generated per job_id
2PIN stored in DB linked to job_id + homeowner_id. Not visible to vendor in advance.
3Vendor arrives on site → asks HO verbally → enters PIN into Vendor Lite Mode app
4POST /api/jobs/:id/pin-verify validates against DB. Timestamp + GPS recorded.
5Success → status = IN_PROGRESS. Admin monitors PIN success/failure ratio in real-time.
⚠3 failures → Admin alert raised. Potential impersonation flag. Admin can freeze job.
